# 数据处理协议（DPA）模板

本文档是公司与外部数据处理者签订**数据处理协议（Data Processing Agreement, DPA）**
的标准模板要点。具体条款以法务审阅后定稿为准。

## 适用情形

任何外部供应商在为我司提供服务过程中**接触到个人信息或客户业务数据**时，
都需要单独或在主合同附件中签订 DPA。例子：

- 云服务商（AWS / 阿里云 / GCP）
- 海外 SaaS（Slack / Google Workspace / OpenAI / Anthropic）
- 数据分析 / BI 外包
- 客服 / 内容审核外包
- 第三方支付 / KYC / 风控

## 必备条款（清单）

| 条款 | 要点 |
|---|---|
| 双方角色 | 明确「我司 = 数据控制者」「对方 = 数据处理者」 |
| 处理范围 | 仅在合同列举的目的、地理范围、子处理者范围内处理 |
| 安全措施 | 加密、访问控制、审计、漏洞披露、SOC 2 / ISO 27001 等级声明 |
| 子处理者 | 列明初始名单；新增子处理者需 30 天内通知，我司可异议 |
| 跨境 | 明确数据存储 / 处理地点；启用 EU SCC / 国家网信办标准合同 |
| 数据主体权利 | 访问 / 更正 / 删除 / 数据可携 / 拒绝自动化决策 |
| 事故响应 | 数据泄露后 72 小时内通知我司 |
| 审计权 | 我司每年至少一次现场或远程审计 |
| 终止 | 合同终止后 30 天内删除或返还所有数据 |

## 数据主体权利 SLA

> **关键 SLA：数据主体（`data_subject`）请求必须在 30 天内完成响应。**

| 请求类型 | 我司响应 SLA | 对方响应 SLA |
|---|---|---|
| 访问 | 15 工作日 | 7 工作日 |
| 更正 | 15 工作日 | 7 工作日 |
| 删除 | 15 工作日 | 30 天内彻底删除（含备份） |
| 数据可携 | 30 天 | 配合提供机器可读格式 |
| 拒绝自动化决策 | 30 天 | 配合提供人工复核 |

数据处理者必须在 SLA 到期前**至少 3 个工作日**反馈处理结果。

## 数据泄露通知 SLA

- 数据处理者发现疑似泄露后 **72 小时内**通知我司
- 通知必须包含：受影响数据类型 / 数量、根因、已采取措施、补救计划
- 我司在 72 小时内向监管机构（GDPR Art. 33）+ 5 个工作日内通知数据主体（按风险）

## 子处理者管理

- 签订时随附「初始子处理者清单」附件
- 新增子处理者：处理者需提前 30 天书面通知；我司有 14 天异议期
- 异议成立则双方协商：要么不增加该子处理者，要么终止合同

## 终止与数据销毁

- 合同终止后 30 天内删除或返还所有数据
- 删除完成后处理者出具《数据销毁声明》
- 涉及备份的，数据处理者声明备份按各自留存策略最终销毁

## 特殊场景

### 模型训练

未经单独书面同意，处理者**不得**将我司提供的数据用于训练自有模型 / 算法。
即使脱敏处理后亦需明确「training opt-out」条款。

### 跨境

参考 `data_compliance.md` 的「跨境数据出境」段。
DPA 中 cross-border 条款应明确选用：

- 欧盟 SCC（最新版）
- 国家网信办「标准合同」（PIPL）
- 或经备案 / 安全评估的特殊路径

### 紧急执法请求

供应商收到所在国执法机关的数据请求时：

- 必须在法律允许范围内立即通知我司
- 仅提供「最小必要」范围的数据
- 提供后 7 个工作日内提交完整请求副本与响应记录
