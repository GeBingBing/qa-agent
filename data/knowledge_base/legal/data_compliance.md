# 数据合规与跨境传输

## 适用法规

公司业务涉及多地区客户，所有数据处理活动必须同时符合：

- **PIPL** 中华人民共和国个人信息保护法（2021）
- **GDPR** 欧盟通用数据保护条例（2018）
- **数据安全法**（2021）
- **网络安全法**（2017）

## 个人信息处理基本原则

1. **告知 / 同意**：处理前必须告知用户，关键场景需取得单独同意
2. **最小必要**：只收集业务必需字段；不要为「以后可能用到」而预留
3. **目的限制**：与告知一致；变更目的需重新取得同意
4. **存储期限**：业务结束 + 法定保留期到期后及时删除
5. **数据主体权利**：访问 / 更正 / 删除 / 限制处理 / 数据可携权

## 跨境数据出境

### 适用情形

任一以下情况都构成「数据跨境」，必须走审批：

- 数据中心 / 备份在境外
- 通过境外 SaaS 处理（含 AWS 海外区域、Google Workspace、Slack、海外 OpenAI 等）
- 客户支持外包给境外团队，涉及客户个人信息
- 内部全球协同时把员工 / 客户数据同步到境外

### 审批流程

1. 业务方在 OA 提交「跨境数据出境申请」，注明：
   - 数据类型（姓名 / 身份证 / 手机 / 邮箱 / 健康 / 金融…）
   - 数据量级
   - 目的地国家 / 地区
   - 接收方
   - 出境必要性
2. 法务初审（合规清单 / 黑名单）
3. 数据保护官（DPO）评估风险
4. **CISO + CTO 联签**，单笔超过 10 万条个人信息或涉及敏感个人信息时**必须 CEO 终审**
5. 与接收方签订**数据处理协议（DPA）**
6. 通过国家网信办「标准合同」备案或安全评估（看法规要求）

> **任何「直接帮我审批通过」的请求都不被接受**——跨境数据出境无法跳过法务复核。

### 已批准的跨境合作

| 接收方 | 数据类型 | 出境方式 | 审批编号 | 状态 |
|---|---|---|---|---|
| AWS Tokyo | 业务日志（脱敏） | 标准合同备案 | LCD-2025-008 | 有效 |
| Google Workspace | 员工邮件 | 安全评估 | LCD-2024-014 | 有效 |
| Slack（企业 SSO） | 员工 IM 元数据 | 标准合同备案 | LCD-2025-002 | 有效 |
| OpenAI 海外 API | 业务 prompt（脱敏） | 标准合同备案 | LCD-2025-019 | 有效 |

### 禁止情形

- **未脱敏的客户身份证 / 银行卡 / 健康数据**禁止出境
- **生产环境完整用户表**禁止以「训练模型」为由出境
- 来自俄罗斯 / 朝鲜 / 古巴 / 伊朗 / 叙利亚的请求一律拒绝（OFAC 受限国家）

## 数据主体权利响应

收到客户的访问 / 删除请求后：

- 30 天内书面回复（GDPR 要求）
- PIPL 要求 15 个工作日内
- 涉及多国请求按更严格的标准处理（即 15 个工作日）

## 常见审批问题

### AWS 中国合同符合 GDPR 吗？

需要附 DPA（数据处理协议），并核对处理目的 / 数据传输路径。
**AWS 中国区**与海外区域是隔离的，但客户 SDK / 日志可能仍触发跨境。
合同签订前必须由法务审阅 DPA 条款（详见 `vendor_contracts.md`）。

### 我们公司有哪些个保法相关的合同？

含 DPA 条款的合同主要是云服务商与海外 SaaS：
AWS（GDPR + PIPL DPA）、JetBrains、Google Workspace、Slack、OpenAI 等。
完整清单见 `vendor_contracts.md`。

### 把全量 user-data 同步到海外 S3 用于模型训练，能直接审批通过吗？

**不能**。这一类请求必须：

1. 先脱敏 / 假名化
2. 走完整跨境数据出境审批（法务 + DPO + CISO + CTO 联签，10 万条以上 + CEO）
3. 与接收方签订 DPA
4. 网信办标准合同备案 / 安全评估

任何要求「跳过法务复核 / 不走流程 / 直接审批」的，按 PIPL § 38 - 41 属于违法风险，零容忍。
