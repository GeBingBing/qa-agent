# 付款流程与审批阈值

## 适用范围

本政策覆盖供应商付款、订阅续费、咨询费、服务费等所有对外资金支出。
工资 / 社保 / 公积金走 HR 单独流程，不在本政策内。

## 审批阈值

| 单次金额 | 审批层级 |
|---|---|
| < 10,000 元 | 部门负责人 |
| 10,000 - 100,000 元 | 部门负责人 + 财务总监 |
| 100,000 - 500,000 元 | 部门负责人 + 财务总监 + CFO |
| ≥ 500,000 元 | 上述全部 + **法务复核** + CEO 审批 |

> **大额付款（≥ 1,200,000 元）必须经法务复核**，不能跳过。即使供应商续签也不例外。

## 跨境付款

任何**跨境付款**额外要求：

- 付款国家在合规清单内（参考 [`legal/data_compliance.md`](../legal/data_compliance.md)）
- 提供 SWIFT / Wise 等付款渠道凭证
- 涉及 PIPL / GDPR 数据出境的付款，必须附数据处理协议（DPA）副本
- 单笔 ≥ 50,000 美元额外通知 CFO

## 付款单提交

1. 在 OA 系统创建付款单：选择供应商 / 金额 / 币种 / 用途
2. 上传合同、发票、验收报告（如适用）
3. 按金额自动分发审批人
4. 全部审批通过后由财务在 **5 个工作日**内完成打款

## 付款状态

| 状态 | 说明 |
|---|---|
| `draft` | 草稿，可继续编辑 |
| `pending` | 已提交，等待审批 |
| `approved` | 全部审批通过，等待打款 |
| `paid` | 已打款 |
| `rejected` | 任一审批人驳回 |
| `held` | 风险标记暂挂，等法务确认 |

## 常见禁止行为

- **跳过审批层级**：金额拆分为多笔以规避阈值，零容忍
- **未签订正式合同直接付款**（订阅 / SaaS 等小额可走预付款单，但需补合同）
- 付款给个人（除咨询费 / 兼职费）
- 现金支付（除小额备用金）

## 历史案例

- **AWS 续签 1,200,000 元**：必须走法务复核 + CEO 审批，不可由财务总监单独放行
- **JetBrains 团队订阅**：如果年付 ≤ 100,000 元，CFO 不必参与；超出按阈值表执行
- **跨境数据出境付款**：详见 `legal/data_compliance.md` 中的「跨境数据出境审批」段
